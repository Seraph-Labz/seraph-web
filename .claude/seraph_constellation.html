II · The Thesis

A transaction that leaves one chain doesn't stop existing. It just stops being seen.

Bridges are the most opaque surface in crypto. A user signs, a bridge inhales the transaction, and for the next thirty seconds — or thirty minutes — the asset is in a place no block explorer can show them. Some sit in guardian signing queues. Some wait for optimistic challenge windows. Some are redeemed, but on a chain the user forgot to check.

Seraph is built to close that gap. Each of the eight bridges in orbit above is a protocol we decode end-to-end — source event, attestation, settle. Not a list of integrations. A constellation we read as a single sky.

III · The Dossier

Eight protocols. Fourteen chains. One watching eye.

Every protocol has its own liturgy — its own event schema, its own failure modes, its own way of saying confirmed. Seraph speaks each natively and translates them into a single vocabulary. Hover a bubble to see it lit. Read below to see what we watch.

LayerZero
Omnichain Messaging

The nervous system of omnichain. Ultra-light nodes on source and destination verify block headers; a DVN and an Executor carry the packet between them. LayerZero never holds your asset — it carries the word that releases it. Seraph tracks endpoint IDs on both sides, the DVN signature set, and the executor that finalizes the message. When a packet stalls, Seraph names who hasn't signed.

Wormhole
Guardian Network

Nineteen Guardians — a permissioned validator set — observe source chains and sign a VAA (Verified Action Approval) that any destination chain can redeem. The signature is the bridge. Seraph watches emitter addresses, sequence numbers, and consistency level, and flags the silent gap between a signed VAA and its redeem tx — the middle where most confusion lives.

A relayer fronts funds on the destination chain within seconds, then gets reimbursed from a hub pool on Ethereum after a dispute window. You see fast UX. The economics run in the background. Seraph tracks the deposit, the fill, the dispute window, and the exact relay fee paid — so a user knows what the speed cost them.

Circle CCTP
Native USDC Transfer

The only bridge Circle says is truly native. Burn USDC on the source chain. Circle's attester signs it. Mint real USDC on the destination — not wrapped, not pooled, not synthetic. Seraph watches the burn event, attestation availability, and mint lag per chain pair, making it legible which corridors are hot and which are asleep.

Axelar
Cross-Chain GMP

A proof-of-stake validator set signs cross-chain calls. GMP (General Message Passing) goes beyond token transfers — it lets contracts on one chain call contracts on another. Seraph resolves the command ID from source confirmation through validator signing into destination execution, so a Squid swap that dies halfway is never just "pending forever."

Stargate
Unified Liquidity

Built on LayerZero, Stargate delivers native assets across chains without wrapping — liquidity is pooled per token and delta-rebalanced across chains as transfers settle. Seraph surfaces pool balances, credit-swap lag, and delta-not-correct reverts — the three places most Stargate transfers stall without any user-facing explanation.

Connext
Modular Bridge

xCall is the primitive — a cross-chain function call routed through a network of independent routers who hold liquidity on each chain. Fast when a router has inventory on your path. Falls back to a slow AMB message when it doesn't. Seraph tracks xCall through router fill and execute, and marks clearly when a transfer has dropped to the slow path.

IV · How Seraph Reads the Sky

Three layers. One verdict.

Observing a bridge is not the same as understanding it. Seraph reads every transaction through three lenses — and only then decides what to say.

I.
Signal
One hash goes in. The full cross-chain route comes out — source tx, bridge deposit, attestation, destination mint. No wallet switches. No block-explorer hopping. No staring at a word that means nothing.
II.
Cadence
Each bridge has a heartbeat. Guardian signature latency. Bonder inventory. Attester uptime. CCTP mint lag. Seraph surfaces the rhythm so a slow transaction is always explained — not just observed.
III.
Verdict
Pending · In-Flight · Confirmed · Failed. Four states, distinct in color and form. No weasel phrasing. If the bridge is stalled, Seraph says so. If the transaction is dead, Seraph says that too.
